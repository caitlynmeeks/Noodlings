# Noodlings Release Strategy

**Status:** Planning Phase
**Target:** Q1 2026 Public Release
**Last Updated:** November 5, 2025

---

## Executive Summary

Noodlings is a hierarchical affective consciousness architecture implementing predictive processing with multi-timescale learning. Rather than competing directly with established platforms, we're pursuing a **multi-pronged strategy**:

1. **noodleMUSH Books** - Entertainment market: "Living Stories" you can inhabit (PRIMARY FOCUS)
2. **B2B Entertainment** - Licensing to studios (Illumination, Pixar, etc.) for character persistence
3. **SillyTavern Integration** - Quick market entry as a backend option for existing ST users
4. **VR Authority Layer** - noodleMUSH as the "brain" for generative real-time VR worlds
5. **Hugging Face Release** - Scientific validation and open-source community building

### The Origin Story

**"Trained on solar power in the rain, with diesel generators as backup."**

This project was trained off-grid on Apple Silicon powered by a solar bank, with diesel generators charging the system during overcast days. When battery reserves ran low, the fridge was unplugged to keep the training running. This isn't VC-funded compute - this is conviction. Every parameter was learned while managing real-world resource constraints, making Noodlings uniquely positioned for efficient, local-first AI.

## Phase 6: Appetite Architecture - Motivational Layer

### The Missing Piece

**Current State (Phase 4):**
Noodlings have rich phenomenal experiences (Broth, Flavors, multi-timescale memory) but lack explicit **motivational architecture** that drives sustained goal-directed behavior. They react to stimuli but don't pursue wants that emerge from personality.

**The Problem:**
How does Brenda orchestrate stories without puppeting characters? How does Mr. Toad decide to pursue motor-cars with manic vitality? How do Noodlings have conflicting desires ("I want to help Ratty, but I also crave adventure")?

**The Solution: Appetite Layer**
An 8-12D motivational space that sits alongside the slow layer, accumulating drives that generate goals, create conflicts, and shape behavior chains.

### Technical Implementation

**Phase 6 Addition to Noodling Architecture:**

```python
class AppetiteLayer(nn.Module):
    """
    Drives goal-directed behavior through accumulating appetites.
    Sits alongside slow layer, influences goal generation.
    """
    def __init__(self, appetite_dim=8, goal_dim=16):
        super().__init__()
        self.appetite_dim = appetite_dim
        self.goal_dim = goal_dim

        # Appetite state (accumulates when unsatisfied)
        self.appetites = mx.zeros((1, appetite_dim))

        # Map from (slow_layer + recent_affect) ‚Üí appetite accumulation
        self.appetite_dynamics = nn.Linear(8 + 16, appetite_dim)

        # Map from (appetites + perception + context) ‚Üí goal activation
        self.goal_generator = nn.Linear(appetite_dim + 5 + 16, goal_dim)

        # Detect conflicts (goals that can't coexist)
        self.conflict_detector = nn.Linear(goal_dim, goal_dim)

    def forward(self, slow_state, fast_state, affect):
        """
        Update appetites and generate goals.

        Args:
            slow_state: 8-D personality/disposition
            fast_state: 16-D immediate context
            affect: 5-D current affect

        Returns:
            goals: 16-D vector of active goal strengths
            conflicts: 16-D conflict tensor (negative = incompatible)
            appetites: 8-D current appetite state
        """
        # Appetites accumulate based on personality and recent affect
        appetite_input = mx.concatenate([slow_state, fast_state], axis=1)
        appetite_change = self.appetite_dynamics(appetite_input)

        # Accumulation rate: 0.1 per timestep when unsatisfied
        self.appetites = mx.clip(
            self.appetites + appetite_change * 0.1,
            0.0, 1.0
        )

        # Generate goals from appetites + current state
        goal_input = mx.concatenate([
            self.appetites,
            affect,
            fast_state
        ], axis=1)
        goals = mx.sigmoid(self.goal_generator(goal_input))

        # Detect conflicts (learned: which goals negatively correlate)
        conflicts = mx.tanh(self.conflict_detector(goals))

        return goals, conflicts, self.appetites

    def satisfy_appetite(self, appetite_idx, amount):
        """
        Called when action satisfies an appetite.
        E.g., learns something new ‚Üí curiosity decreases temporarily
        """
        self.appetites[0, appetite_idx] = max(
            0.0,
            self.appetites[0, appetite_idx] - amount
        )

    def get_top_goals(self, goals, k=3):
        """Return top-k most active goals for LLM context."""
        goal_names = [
            "explore_environment", "seek_social_connection",
            "demonstrate_competence", "pursue_novelty",
            "ensure_safety", "gain_status", "seek_comfort",
            "maintain_autonomy", "help_friend", "avoid_consequences",
            "restore_reputation", "learn_skill", "impress_others",
            "solve_problem", "express_emotion", "achieve_goal"
        ]

        top_indices = mx.argsort(goals[0])[-k:]
        return [(goal_names[i], float(goals[0, i])) for i in top_indices]
```

def forward(self, affect_input, context):

    h_fast, c_fast = self.fast_lstm(affect_input, self.h_fast, self.c_fast)
    h_med, c_med = self.medium_lstm(h_fast, self.h_med, self.c_med)

    joint = mx.concatenate([h_fast, h_med, h_slow, goals], axis=1)
